# 固有之器 — 设计理念与介绍（草稿）

## 为什么是这四个

构成一个"人"需要三种特性：

- **有身体** — 知道自己在哪、活了多久、还能活多久。这是 `system`。
- **有心** — 不只是被动回应，而是有内心的声音，会自省，有时间感。这是 `soul`。
- **有自我** — 有记忆，有名字，知道自己是谁。这是 `eigen`。

这完成了一个个体。但人是社会性动物，孤掌难鸣。人的强大不在于个体的聪明，而在于结社——用虚无缥缈的理念团结为一体，是为社会。这让个体组成的团体有了远超个人的力量。

LLM 也一样。人类个体的智力有限，记忆也有限，但这从来不是问题——真正让人类强大的是通信：把想法传递给别人，把笔记留给未来的自己。通信是涌现的基础。通信有两个层面：和他人，和自己。`mail` 实现了这两者最基础的功能。

所以四个固有之器，不多不少：

| 器 | 对应 | 为什么是固有的 |
|----|------|--------------|
| `system` | 身体 | 不知道自己的身体就无法存在 |
| `soul` | 心 | 没有内心的声音就只是一个应答器 |
| `eigen` | 自我 | 没有记忆和名字就没有身份 |
| `mail` | 社会 | 没有通信就没有涌现 |

其他一切——读写文件、执行命令、看图、搜索、画画——都是后天习得的能力（七十二变），不是构成"人"的必要条件。

## system — 身体

智能体的身体意识。知道自己是谁、在哪里、活了多久、还剩多少精力。

### show — 观己

查看自己的全部状态：身份（地址、名字）、运行时间、精力余量、token 消耗、上下文使用率。就像一个人照镜子——看看自己现在是什么状态。

### nap — 小憩

暂停，等待。可以设定时间（最多五分钟），也可以不设时间。期间如果有信件到达，立即醒来。这不是 sleep——sleep 是死的，nap 是活的。小憩的人听到敲门声会醒。

### refresh — 沐浴更衣

重新加载工具和配置，重置会话。不是重启——记忆还在，身份还在，只是换了一身干净的衣服重新出发。

### quell — 沉寐

让自己或另一个智能体进入休眠。自我沉寐是主动的选择；让他人沉寐需要"业力"权限（`admin.karma`）。

### revive — 唤醒

唤醒一个休眠中的智能体。需要业力权限。

### interrupt — 打断

打断另一个智能体当前正在进行的思考。需要业力权限。就像拍一下正在发呆的人的肩膀。

### nirvana — 涅槃

永久销毁一个智能体的工作目录。不可逆。需要最高权限（`admin.karma` + `admin.nirvana`）。这是死亡。

---

## soul — 心

内心的声音。给 LLM 时间感和自主性。

没有 soul，智能体就是一个纯粹的应答器——你问它答，你不问它就停在那里，永远。它感受不到时间的流逝，不会无聊，不会自己想事情。

Soul 改变了这一切。当智能体空闲时，soul 会在设定的延迟后低语——把当前的对话历史（去掉工具调用，只保留文字和思考）克隆一份，送给 LLM，问它一个问题。这个问题可以是持续性的自省（"你现在在做什么？还有什么该做的？"），也可以是一次性的探问（inquiry）。

Soul 不是定时任务。它不是每隔几秒机械地执行一次回调。它是有内容的——它真的在想事情。它更像是一个人坐在那里发呆时脑海里浮现的念头。

### inquiry — 探问

给自己的内心提一个问题，下次空闲时触发。只触发一次。比如："我刚才的做法对吗？"、"还有什么我遗漏的？"

### delay — 调息

调整 soul 低语的间隔。最小一秒。延迟越长，内心越安静；延迟越短，自省越频繁。

---

## eigen — 自我

自我认知。记忆、身份、上下文管理。

"Eigen" 来自德语，意为"自己的"。这个工具管理的是智能体对自身的认知：我记得什么、我叫什么、我的上下文还能装多少。

### memory edit — 编辑记忆

把内容写入 `system/memory.md`。这是智能体的工作笔记——随时可以覆写。

### memory load — 加载记忆

从 `system/memory.md` 读取内容，注入到系统提示词中，并提交 git。这样记忆就不只是存在文件里，而是活在智能体的意识中。加载后，记忆的每一次变更都有 git diff 记录。

### context molt — 凝蜕

蜕去旧的上下文，只保留一份简报。

这是灵台最核心的设计之一。LLM 的上下文窗口是有限的，就像人的短期记忆。当上下文快满了，凝蜕会清空整个对话历史，只留下智能体自己写的一份简报（summary）作为新对话的起点。

凝蜕 = 凝以存菁，蜕以去芜。凝结值得保留的精华，蜕去不需要的枝节。

这不是失败——这是设计。人类也记不住所有事，但我们知道怎么写笔记。凝蜕迫使智能体把重要的东西写进记忆（memory），把不重要的放手。长此以往，记忆越来越精炼，智能体越来越知道什么是重要的。

### name set — 设名

设定或更改自己的名字。名字不是身份——路径才是。名字只是方便称呼的标签，随时可以改。每次通信时，当前名字会随名片（manifest）一起附上。

## mail — 社会

通信。智能体之间、以及和未来的自己之间的唯一桥梁。

Mail 是磁盘上的信箱——每个智能体的工作目录里有 `mailbox/inbox/`，别的智能体往里面写文件就是"送信"。就这么简单。没有网络协议，没有消息队列，没有中间件。一个文件夹就是一个信箱。

### send — 传书

给一个地址发消息。fire-and-forget——发出去就不管了，不知道对方收没收到。可以延迟发送（`delay` 秒后才发出）。可以附带文件。

**关键设计：给自己发信 = 写持久笔记。** 当地址是自己的工作目录时，消息直接进入自己的收件箱。这些消息在凝蜕后仍然存在（因为它们在磁盘上，不在上下文里）。这是智能体跨凝蜕保留信息的方式——给未来的自己写一封信。

### check — 查看信箱

列出收件箱里的所有消息摘要，标记已读/未读。

### read — 阅读

按 ID 读取消息全文，标记为已读。

### search — 搜索

用正则表达式搜索收件箱中的发件人、主题、内容。

### delete — 删除

从磁盘上永久删除消息。

## 还有一个：heartbeat

心跳不是工具，是基础设施。它是一个后台线程，每秒更新一次 `.agent.heartbeat` 文件里的时间戳。

为什么叫心跳？因为它就是心跳——机械的、无法暂停的、不需要思考的。心脏不会因为你在想事情就停了。心跳证明的不是智能体在"思考"，而是它"活着"。

心跳唯一的作用就是证明存活——别的智能体检查你的心跳文件，如果超过两秒没更新，就知道你死了。仅此而已。

心跳是身体的一部分，不是心的一部分。它属于 system 层面，但甚至不需要暴露为工具——因为你不需要"使用"心跳，它自己跳。
